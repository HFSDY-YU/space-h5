# 澳琴空间预约系统 H5 移动端 UI 设计方案

本文档基于 `space-h5/功能清单.md` 与 `space-h5/ui设计/` 下的三张参考图制定，用于指导 H5 第一版界面设计与后续前端实现。

参考图：

- `登录页面.png`
- `主页.png`
- `我的页面.png`

## 1. 设计定位

H5 手机端不是 PC 后台的缩小版，而是面向老师、学生、管理员在移动场景下的高频操作入口。

项目主题来自根目录 `README.md`：澳琴空间预约系统是面向澳琴国际教育大学城的空间资源预约、审核与管理系统。移动端视觉与文案应围绕“校园空间资源”“预约审核”“公开占用查询”展开，不做泛后台风格，也不做营销型首页。

核心目标：

- 学生：快速查看房间占用和预约公示，不出现预约、审核、管理入口。
- 老师：快速查询可用房间，提交单次或长期预约，查看个人预约状态和取消待审核预约。
- 管理员：快速处理待审核预约，查看房间状态，进行轻量启用/停用管理。

第一版设计优先保证：

- 登录清晰。
- 首页查房高效。
- 预约流程短。
- 我的预约状态可读。
- 管理员审核动作明确。
- 三类角色入口隔离。

## 2. 视觉方向

### 2.1 关键词

- 清爽
- 高效
- 澳琴校园空间感
- 资源预约秩序感
- 蓝白主调
- 轻卡片
- 状态一眼可辨
- 手机触控友好

### 2.2 从参考图继承的视觉语言

登录页：

- 顶部使用浅蓝空间插画或空间场景背景。
- 主标题大字突出“澳琴空间预约系统”。
- 副标题使用“澳琴国际教育大学城空间资源预约平台”或“空间预约 · 审核管理 · 占用查询”这类贴近项目定位的短文案。
- 表单放在白色大圆角面板中。
- 输入框使用浅灰边框、左侧线性图标。
- 主按钮使用饱和蓝色。
- 协议、隐私、忘记密码使用蓝色文字链接。

首页：

- 顶部强蓝渐变区域承载页面标题和消息入口。
- 搜索框和筛选按钮紧贴标题下方，便于单手操作。
- 日期切换和“仅看有空闲”形成查询控制区。
- 首页不展示空闲、审核中、占用数量统计组件。
- 首页不展示房间类型分类组件。
- 房间占用以矩阵表格展示：行是房间，列是时段。
- 房间时段状态只展示空闲、审核中、占用三类。
- 首页矩阵房间列和时段状态块都可以点击进入详情。
- 底部 TabBar 固定，图标 + 文案。

我的页：

- 顶部浅蓝空间背景，展示头像、姓名、角色。
- 内容区使用白色大圆角卡片分组。
- 列表项使用圆形浅色图标、标题、辅助状态、右箭头。
- 退出登录按钮独立放置，红色文字，避免误触。

### 2.3 色彩规范

主色：

- 主蓝：`#1677FF`，用于主按钮、选中态、关键链接、Tab 选中。
- 深文本：`#0F1F3D`，用于标题和重要文字。
- 次级文本：`#69758A`，用于说明、位置、容量等辅助信息。
- 弱文本：`#9AA4B2`，用于占位符、未绑定状态。
- 页面底色：`#F5F8FE`。
- 卡片底色：`#FFFFFF`。
- 分割线：`#E8EEF7`。

状态色：

- 空闲：绿色 `#12B76A`，浅底 `#EAF8F0`。
- 审核中：琥珀色 `#F59E0B`，浅底 `#FFF7E6`。
- 占用：橙色 `#FF6B2C`，浅底 `#FFF1E8`。
- 驳回/失败：红色 `#F04438`，浅底 `#FEECEC`。

说明：

- 首页和房间详情中的房间时段状态只使用空闲、审核中、占用。
- 停用、禁用、删除等管理态不作为普通用户房间占用状态展示；普通用户列表应过滤不可用房间，管理员房间管理中可用开关或管理字段单独表达。

### 2.4 字体与圆角

- 页面标题：28-32px，加粗。
- 区块标题：18-22px，加粗。
- 列表主标题：16-18px，中粗。
- 辅助文字：13-15px。
- 底部 Tab 文案：12px。
- 页面卡片圆角：20-24px。
- 输入框、状态块、按钮圆角：12-16px。
- 小标签圆角：999px。

## 3. 信息架构

### 3.1 角色菜单

学生：

- 首页：房间查看、占用矩阵、预约公示和房间详情入口。
- 消息：右上角铃铛入口，不进入底部 TabBar。
- 我的：个人信息、账号安全、协议、退出登录。

老师：

- 首页：房间查看与快速预约。
- 预约：单次预约为默认模式，长期预约作为高级模式折叠进入。
- 我的预约：预约记录、详情、取消。
- 消息：右上角铃铛入口。
- 我的：个人信息、账号安全、退出登录。

管理员：

- 首页：房间查看。
- 审核：待审核、取消待审核。
- 房间：轻量房间管理，启用/停用。
- 消息：右上角铃铛入口。
- 我的：个人信息、账号安全、退出登录。

### 3.2 底部 TabBar 方案

学生版：

```text
首页 / 我的
```

老师版：

```text
首页 / 预约 / 我的预约 / 我的
```

管理员版：

```text
首页 / 审核 / 房间 / 我的
```

说明：

- TabBar 根据角色动态生成。
- 非权限功能不展示入口，不做点击后提示无权限。
- 首页始终是房间查看，保证三类角色认知一致。
- 学生的占用查看合并在首页，不再单独提供“占用”Tab。
- 消息统一放在页面右上角铃铛入口，不作为学生底部 Tab。

## 4. 页面设计

### 4.1 登录页

页面目标：

- 支持统一账号登录。
- 支持记住登录状态。
- 支持忘记密码。
- 保留用户协议和隐私政策确认。
- 登录后根据后端返回的用户角色和权限自动进入对应首页。

页面结构：

1. 顶部状态栏安全距离。
2. 浅蓝空间场景插画。
3. 主标题“澳琴空间预约系统”。
4. 副标题“澳琴国际教育大学城空间资源预约平台”。
5. 白色圆角登录面板。
6. 账号输入框。
7. 密码输入框。
8. 验证码输入框。
9. 记住登录状态 + 忘记密码。
10. 登录按钮。
11. 协议勾选。

交互规则：

- 未勾选协议时点击登录，Toast 提示“请先阅读并同意用户协议和隐私政策”。
- 页面不提供老师、学生、管理员角色分段控件。
- 账号角色和权限以后端 `/getInfo` 返回结果为准，前端只根据权限生成 TabBar 和页面入口。
- 密码可切换明文/密文。
- 验证码失败后刷新验证码。
- 登录成功后自动进入对应角色首页。

### 4.2 首页：房间查看

页面目标：

- 快速查看当天房间可用情况。
- 支持搜索、筛选、日期切换、房型切换。
- 老师可从空闲时段发起预约。
- 学生只能查看详情。
- 管理员可进入房间轻管理或审核。
- 首页同时承担学生“占用查看”职责，不再单独拆出占用 Tab。

页面结构：

1. 顶部蓝色渐变头图。
2. 页面标题“澳琴空间预约”或“房间查看”。
3. 副标题“查看空间占用情况，快速发起预约”。
4. 消息铃铛，未读红点。
5. 搜索框：房间名称 / 编号。
6. 筛选按钮。
7. 日期切换卡片：上一天 / 日期 / 下一天 / 今天。
8. “仅看有空闲”按钮。
9. 房间占用矩阵。
10. 底部 TabBar。

不展示：

- 空闲、审核中、占用数量统计组件。
- 房间类型分类按钮组。

首页卡片：

- 首页主体以占用矩阵为主，不额外展示独立房间卡片列表。
- 矩阵第一列的房间信息区域可点击进入房间详情页。
- 老师点击空闲时段可进入房间详情后发起预约，学生点击只能查看占用详情，管理员点击可进入详情或管理动作。

占用矩阵：

- 第一列：房间信息。
- 后续列：标准时段。
- 房间信息包含：房间编号、房间类型、位置、容量。
- 时段状态块包含：图标 + 状态文字。

状态块：

- 空闲：绿色勾选图标，点击查看详情；老师可在详情页发起预约。
- 审核中：琥珀色时钟图标，点击查看该时段审核中说明；不展示敏感审核信息。
- 占用：橙色日历图标，点击查看已通过预约公示和场次明细。

交互规则：

- 点击矩阵第一列房间信息区域，进入 `/rooms/:roomId`。
- 点击时段状态块，进入房间详情并定位到对应日期与时段。
- 空闲状态不直接在首页提交预约，避免误触；预约动作放在详情页或预约页确认。

移动适配：

- 375px 宽度下矩阵必须严格限制在屏幕范围内，不支持左右滚动。
- 矩阵列宽使用百分比布局，房间列与三个时段列共同占满容器宽度。
- 首页顶部搜索区不随矩阵上下滚动移动。
- 占用矩阵在剩余可视区域内上下滚动，不能遮挡底部 TabBar。

### 4.3 筛选抽屉

触发：

- 首页点击“筛选”。

内容：

- 楼栋。
- 楼层。
- 房间类型。
- 容量范围。
- 设备：投影、白板、音响等。
- 时段状态：空闲、审核中、占用。

底部按钮：

- 重置。
- 确定。

交互规则：

- 使用底部弹出层或右侧抽屉，移动端优先底部弹层。
- 筛选项尽量使用标签组，不使用复杂树形控件。

### 4.4 房间详情页

页面目标：

- 查看房间基础信息、设备、指定日期占用明细。
- 老师可对空闲时段快速预约。
- 学生只查看。
- 管理员可轻量修改启用/停用。

页面结构：

1. 顶部导航栏：返回 + 房间编号。
2. 房间概览卡：名称、类型、位置、容量、状态。
3. 设备标签：投影、白板、音响等。
4. 日期选择条。
5. 时段占用列表。
6. 预约公示卡片列表。
7. 底部固定操作区。

操作区：

- 学生：无主按钮，只展示“刷新占用”。
- 老师：空闲时段展示“预约该房间”。
- 管理员：展示“启用/停用”轻管理按钮。

### 4.5 常规预约页

页面目标：

- 老师在手机端完成单次预约申请。

页面结构：

1. 导航栏：常规预约。
2. 房间选择卡。
3. 日期选择。
4. 时段选择：上午、下午、晚间。
5. 预约主题。
6. 预约用途。
7. 预约人数。
8. 详细备注。
9. 冲突校验提示区。
10. 底部固定提交按钮。

交互规则：

- 从首页空闲状态进入时，自动带入房间、日期、时段。
- 未选择房间、日期、时段时禁用提交。
- 提交中按钮 loading，防重复点击。
- 后端返回冲突时保留表单输入，并高亮冲突时段。

### 4.6 预约页：长期预约轻量模式

页面目标：

- 支持老师提交长期固定排课预约，但移动端第一版必须降低复杂度。

设计调整：

- 第一版不做独立复杂的长期预约大页面。
- 老师底部 Tab 使用“预约”，页面默认是单次预约。
- 长期预约作为“高级预约模式”放在预约页内，通过分段控件切换。
- 复杂批量维护、每日固定大范围生成等能力优先保留给 PC 端或第二版 H5。

页面结构：

1. 预约类型：单次预约 / 长期预约。
2. 长期模式：每周固定 / 自定义多日期。
3. 房间选择。
4. 日期范围或多日期选择。
5. 周几与时段选择。
6. 场次预览，默认折叠，仅展示“将生成 X 个场次”。
7. 预约主题、用途、人数、备注。
8. 底部提交。

交互重点：

- 长期预约必须显示“将生成 X 个场次”。
- 场次预览支持折叠，避免页面过长。
- 冲突结果按场次展示，第一版只允许返回修改，不在手机端做复杂逐项编辑。
- 每日固定先不作为默认入口，避免在手机端生成大量场次造成理解和审核成本过高。

第一版建议：

- 先实现单次预约。
- 长期预约只保留每周固定和自定义多日期两个轻量模式。
- 每日固定、批量多规则叠加、冲突场次逐项删除作为第二阶段。

### 4.7 我的预约页

页面目标：

- 老师查看个人所有预约状态，支持详情和取消。

页面结构：

1. 顶部标题“我的预约”。
2. 状态分段：全部、待审核、已通过、已驳回、已取消、已结束。
3. 搜索或日期筛选。
4. 预约卡片列表。

预约卡片内容：

- 状态标签。
- 预约主题。
- 房间与时段摘要。
- 预约类型：单次 / 长期。
- 申请时间。
- 审核意见摘要。

卡片操作：

- 查看详情。
- 待审核状态可取消。
- 已驳回可查看原因。

### 4.8 预约详情页

页面目标：

- 展示主预约和所有场次明细。

页面结构：

1. 状态头部：状态、预约编号、提交时间。
2. 主信息：主题、用途、人数、备注、申请人。
3. 场次列表：房间、日期、时段、场次状态。
4. 审核记录：审核人、时间、意见。
5. 底部操作。

操作：

- 老师：待审核可取消。
- 学生：只读。
- 管理员：待审核可通过/驳回。

### 4.9 管理员审核页

页面目标：

- 管理员在手机端快速处理预约审核。

页面结构：

1. 顶部标题“预约审核”。
2. 分段：待审核 / 取消待审核 / 审核记录。
3. 筛选：预约类型、日期、申请人、房间。
4. 审核卡片列表。

审核卡片内容：

- 申请人、角色。
- 预约主题。
- 房间和时段摘要。
- 场次数量。
- 冲突提示。
- 提交时间。

操作：

- 通过。
- 驳回。
- 查看详情。

长期批量预约：

- 卡片显示“共 X 个场次”。
- 详情页支持逐条通过/驳回。
- 批量通过前二次确认。
- 批量驳回必须填写原因。

### 4.10 管理员房间轻管理页

页面目标：

- 管理员移动端查看房间并快速启用/停用。

页面范围：

- 查看房间列表。
- 查看房间详情。
- 修改启用/停用状态。

不做：

- 批量导入。
- 永久删除。
- 复杂设备维护。
- 完整后台 CRUD。

原因：

- 移动端只保留应急管理能力，复杂维护仍回 PC 端。

### 4.11 消息页：右上角入口

页面目标：

- 展示预约提交、审核通过、审核驳回、取消审核等消息。
- 消息页不进入学生底部 TabBar，由首页或我的页右上角铃铛进入。

页面结构：

1. 顶部标题“消息”。
2. 分段：全部 / 未读。
3. 消息列表。

消息卡片：

- 消息类型图标。
- 标题。
- 简短内容。
- 时间。
- 未读标记。

第一版说明：

- 若后端暂未提供独立消息接口，可先基于预约状态变化生成本地/列表提示。
- 后续接入 `space_message` 或通知接口后再改为服务端消息源。

### 4.12 我的页

页面目标：

- 展示个人身份、账号安全、协议和退出登录。

页面结构沿用参考图：

1. 顶部浅蓝空间背景。
2. 标题“我的”。
3. 消息铃铛。
4. 头像、姓名、角色。
5. 账号与安全卡片。
6. 其他卡片。
7. 退出登录按钮。
8. 底部 TabBar。

角色显示：

- 学生：蓝色学士帽图标。
- 老师：绿色书本或讲台图标。
- 管理员：橙色盾牌或管理图标。

账号与安全：

- 个人信息。
- 修改密码。
- 绑定手机。
- 绑定邮箱。

其他：

- 关于我们。
- 使用协议。
- 隐私政策。

## 5. 组件规范

### 5.1 通用页面壳

用于：

## 9. 实现注意事项

- 页面背景不要使用单一纯白，建议 `#F5F8FE` + 白色卡片。
- 不要把 PC 端表格直接搬到手机端，除首页占用矩阵外，列表以卡片为主。
- 首页矩阵第一列房间信息必须支持点击查看详情。
- 首页矩阵不支持左右滚动，必须通过列宽、字号和状态块尺寸适配 375px 宽度。
- 首页和房间详情的房间时段状态只展示空闲、审核中、占用。
- 底部固定按钮必须处理 iPhone 安全区。
- 所有确认类动作必须二次确认。
- 驳回类动作必须填写原因。
- 表单失败后保留输入。
- 角色入口必须在前端隐藏，同时后端权限继续兜底。
- 公开页面不得展示后端已清理的内部字段。
